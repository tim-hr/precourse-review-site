Classes: state and behavior

In object-oriented programming, a class is a template for creating objects, providing initial values for each object's state (via the constructor) and implementations of the object's behavior (via methods).

Remember the definition of the spreadsheet's data model in X-Cell? Here's a simplified version to illustrate.

  • The constructor initializes the object's state.
  • The sheet's state is in this.data.
  • State can be read and mutated (changed) at specific locations in the sheet by the two methods.
  • Being able to read and mutate the state of a single cell at a time is the only behavior that the model object has.
  • The new keyword calls the constructor function and returns a newly-created instance of the model object.
  • The new instance has the same properties as the original model object.
  • That happens through prototypal inheritance (remember reading about how property lookup happens via the prototype chain)?)
